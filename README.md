# React 19 · TanStack · shadcn + Base UI · Lenis + GSAP

A production-ready starter template. Not a bare skeleton — it ships a working
demo route that proves the whole stack (smooth scroll, scroll animation,
routing, data fetching, both UI libraries) wires together correctly.

## Stack

| Concern          | Choice                                                       |
| ---------------- | ------------------------------------------------------------ |
| Framework        | React 19 + TypeScript (strict, no `any`)                     |
| Build tool       | Vite 8                                                        |
| Routing          | TanStack Router (file-based, type-safe route tree)           |
| Server state     | TanStack Query                                               |
| UI — Radix based | shadcn-ui (Tailwind v4, `new-york` style)                    |
| UI — MUI Base    | Base UI (`@base-ui/react`)                                   |
| Smooth scroll    | Lenis                                                        |
| Scroll animation | GSAP core + ScrollTrigger (both free, incl. commercial)      |
| Linting          | ESLint 10 flat config + `typescript-eslint` + React Hooks    |

## Getting started

```bash
pnpm install
pnpm dev        # start the dev server (http://localhost:5173)
```

Other scripts:

```bash
pnpm build      # type-check (tsc -b) + production build
pnpm preview    # preview the production build
pnpm lint       # ESLint
pnpm lint:fix   # ESLint with --fix
pnpm typecheck  # tsc -b --noEmit
```

## Two UI libraries — who does what

shadcn-ui and Base UI both provide UI primitives and their scopes overlap, so
the split is deliberate to avoid style/behaviour conflicts:

- **shadcn-ui** (`src/components/ui/`) — Radix-based, copied into the repo and
  owned by us. **Default choice** for anything shadcn already offers (Button,
  Card, Dialog, …). Styled with the Tailwind tokens in `src/index.css`.
- **Base UI** (`src/components/base-ui/`) — the successor to MUI Base;
  unstyled, accessible primitives. Reach for it only for primitives shadcn/Radix
  **doesn't** cover (e.g. `Slider`), and style each part with the **same**
  Tailwind tokens so both libraries look identical.

Rule of thumb: **prefer shadcn; add a Base UI wrapper only to fill a gap.** Keep
the folders separate so it's always obvious which library a component comes from.

## How Lenis and GSAP are synced

The classic mistake is running two `requestAnimationFrame` loops — one for Lenis,
one for GSAP — which drift apart. Here GSAP's ticker is the **single** loop and
Lenis is created with `autoRaf: false`. See
[`src/providers/lenis-provider.tsx`](src/providers/lenis-provider.tsx):

```ts
const lenis = new Lenis({ autoRaf: false })

lenis.on('scroll', () => ScrollTrigger.update())

gsap.ticker.add((time) => {
  lenis.raf(time * 1000) // Lenis wants ms; the ticker reports seconds
})

gsap.ticker.lagSmoothing(0)
```

`LenisProvider` wraps the app once (`main.tsx`); consume the instance anywhere
with `useLenis()` (e.g. `lenis.scrollTo('#reveal')` in the hero CTA).

### Only free GSAP is used

`gsap` (core) and `ScrollTrigger` only — both free since 2024, including
commercial use. **No Club GreenSock plugins** (`SplitText`, `MorphSVG`,
`DrawSVG`, `MotionPath`, …). If an effect would normally reach for one, solve it
with core + ScrollTrigger or a free alternative (e.g. `split-type` instead of
`SplitText`).

## The demo route

[`src/routes/index.tsx`](src/routes/index.tsx) renders three full-height
sections:

1. **Hero** — parallax gradient blob (`useParallax`) + a Lenis `scrollTo` CTA.
2. **Reveal** — shadcn Cards that fade/slide in on scroll (`useScrollReveal`,
   GSAP ScrollTrigger).
3. **Toolkit** — a Base UI `Slider`, shadcn `Card`/`Button`, and a list rendered
   from TanStack Query.

Delete `features/scroll-demo/` (and simplify the route) once you've used it as a
reference.
